Modelo relacional vs estructuras No Relacionales
En el modelo relacional, los datos se guardan en tablas 
estructuradas (filas y columnas). Cada tabla tiene un esquema 
fijo (nombre, edad, dirección, etc.). Las relaciones entre tablas se 
hacen con claves (como ID de cliente).
Este modelo funciona muy bien cuando los 
datos son estructurados, predecibles y las 
relaciones están bien definidas.
Este modelo funciona muy bien cuando los 
datos son variables, crecen rápido y las 
relaciones no siempre están definidas.
Las bases de datos NoSQL introducen estructuras de datos más 
flexibles y diversas. No imponen un esquema fijo, cada registro 
(documento, fila o nodo) puede tener un formato distinto o atributos 
variables. Esto las hace ideales para:
• datos semiestructurados (JSON, logs, sensores)
• esquemas que cambian con el tiempo
• sistemas distribuidos que necesitan escalar horizontalmente
NoSQL no reemplaza al modelo 
relacional. Lo complementa, según el tipo 
de datos y de problema.

--- DIAPOSITIVA / PÁGINA 15 ---
No hay una única base de datos ideal. Hay que 
mirar qué tipo de datos manejamos, cómo se 
relacionan, y a qué escala vamos a operar.
2026
¿Qué debemos analizar antes de elegir una base de datos?
Estructura de los datos
• ¿Esquema fijo y homogéneo? → Relacional
• ¿Flexible, variable, en evolución? → NoSQL
Relaciones entre datos
• ¿JOINs frecuentes y complejos? → Relacional
• ¿Relaciones densas o jerárquicas simples? → NoSQL 
(grafos o documentos)
Volumen y crecimiento esperado
• ¿Escenario moderado? → Relacional
• ¿Alta escala o distribución global? → NoSQL

2026
Más allá de los datos: operaciones, consistencia y rendimiento
Tipo de consultas
• SQL: consultas complejas, reportes ad-hoc → Relacional
• Accesos simples, patrones definidos → NoSQL
Transacciones y consistencia
• ACID estricto (bancos, contabilidad) → Relacional
• BASE o eventual consistency (apps web, redes) → NoSQL
Escalabilidad horizontal
• Miles de usuarios, millones de operaciones → NoSQL 
distribuido
• Cargas moderadas con seguridad transaccional → Relacional
