BASE DE DATOS DOCUMENTALES
Una base de datos documental 
es un sistema NoSQL diseñado 
para almacenar, recuperar y 
gestionar información 
semiestructurada, típicamente 
en formatos JSON o BSON. A 
diferencia de las relacionales, no 
requieren esquemas rígidos, 
permitiendo flexibilidad en los 
datos y un alto rendimiento para 
aplicaciones modernas. Son 
ideales para Big Data, catálogos, 
sistemas de gestión de 
contenidos y perfiles de usuario.

Cuando usar base documental?
cuando la flexibilidad y la 
velocidad de desarrollo son más 
importantes que la rigidez de una 
tabla
1. Cuando tus datos cambian seguido (Esquema flexible) Si 
estás lanzando un prototipo o una Startup y no sabés qué 
datos vas a necesitar mañana. Ejemplo: Un catálogo de 
productos de e-commerce. Un celular tiene "memoria RAM" , 
pero una remera tiene "talle" y "color" . En JSON, guardás cada 
uno con sus propias claves sin romper nada.

Desnormalización
La desnormalización en bases de datos 
documentales (como MongoDB) es una 
técnica de modelado que integra datos 
relacionados dentro de un mismo 
documento (embedding) para evitar 
uniones (joins) complejas. A diferencia de 
los sistemas relacionales, aquí se prioriza 
la velocidad de lectura y la eficiencia de 
las consultas sobre la integridad 
referencial, duplicando información 
estratégicamente.

--- DIAPOSITIVA / PÁGINA 16 ---
Consultas simples para recuperar datos
• db.contactos.find() 
Esto devuelve todo el contenido. Es la consulta más básica y compacta.
• Si querés verlo más legible con identación y saltos
db.contactos.find().pretty()
• Consultar por un campo simple: 
db.contactos.find({ nombre: "Carlos Gómez" }) 
MongoDB busca documentos donde ese campo coincida.
• Recuperar por un campo dentro de un objeto anidado:
db.contactos.find({ "direccion.ciudad": "Morón" })
• Recuperar dentro de un array simple:
db.contactos.find({ etiquetas: "cliente" })
MongoDB busca automáticamente dentro del array.
• Recuperar dentro de un array de objetos:
db.contactos.find({ "telefonos.tipo": "trabajo" })

--- DIAPOSITIVA / PÁGINA 18 ---
Eliminación de registros
deleteOne → elimina un documento 
deleteMany → elimina varios documentos 
filtro {} → elimina todos los registros 
drop() → elimina la colección completa
db.contactos.deleteOne({ nombre: "Carlos Gómez" })
MongoDB busca el primer documento que cumpla la condición y lo elimina. 
db.contactos.deleteMany({ etiquetas: "cliente" })
Esto borra todos los documentos que coincidan.
db.contactos.deleteMany({}) MongoDB elimina todos los 
documentos de la colección, porque el filtro vacío coincide con todos. Esto 
no borra la colección, solo su contenido.
db.contactos.drop() Eso sí elimina la colección entera.

--- DIAPOSITIVA / PÁGINA 19 ---
Eliminación de registros seleccionados
deleteOne() siempre elimina el primer documento que coincide con el filtro.
Para seleccionar otro documento coincidente en el filtro, se debe identificar el 
documento de forma única y borrar por ese identificador. Cada documento tiene 
un _id.
Primero se busca: 
db.contactos.find({ ciudad: "Morón" })
Tomás el _id del segundo documento y luego borrás:
db.contactos.deleteOne({ _id: ObjectId("...") })
Eso es lo más preciso porque cada documento tiene un identificador único.
